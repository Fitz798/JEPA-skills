# v1.3.0 Roadmap

This document defines the next delivery milestone for the `jepa-research` skill after `v1.2.1`.

## Release Position

`v1.2.1` closed the research_qa citation-hit gap (80% → 100%) and proved that metadata-aware retrieval + evidence-binding enforcement works. The quality bottleneck has shifted:

- **research_qa** is now saturated at 100%/100% on the current 10-prompt slice
- **reproduction_plan** and **debug_triage** are stuck at 90% executability/debug-hit
- **writing_citation** is at 90% factual correctness
- **long-tail metadata** covers only 22/148 papers (14.9%)

`v1.3.0` should shift focus from QA retrieval quality to (a) making the metadata layer broad enough to support all four modes equally, and (b) closing the remaining 10pp gaps in reproduction, debug, and writing modes.

## Release Goals

1. Expand long-tail metadata coverage from 22/148 to at least 80/148 papers for modality, task, and evidence_level.
2. Increase confirmed core paper code_id links from 3/11 to at least 6/11.
3. Improve reproduction_plan executability from 90% to ≥95% and debug_triage debug-hit from 90% to ≥95%.
4. Improve writing_citation factual correctness from 90% to ≥95%.
5. Maintain research_qa at 100%/100% (no regression).

## Non-Goals

- Full manual annotation of all 148 papers (aim for ~54% coverage, not 100%).
- Automated PDF parsing or NLP-based metadata extraction (save for v1.4.0).
- New eval prompts beyond the existing 40-prompt set.
- External API dependencies for metadata enrichment.

## Success Criteria

`v1.3.0` is complete when all of the following are true:

1. `metadata_completeness_report.json` shows modality/task/evidence_level coverage ≥ 54% (80/148).
2. At least 6/11 core papers have a confirmed local `code_id`.
3. Reproduction executability ≥ 95% on the 10-prompt slice.
4. Debug hit ≥ 95% on the 10-prompt slice.
5. Writing citation factual correctness ≥ 95% on the 10-prompt slice.
6. Research QA remains at 100%/100% (regression gate).

## Workstreams

### W1. Long-Tail Metadata Expansion

Objective: bring enough papers into the annotated set that retrieval quality is uniformly good across modes, not just for the core 11.

Current state: 22/148 papers have modality/task/evidence_level. Core (11) is complete; long-tail only has 11 additional annotated papers — mostly the ones added during the v1.2.0 RQ05/RQ07 targeted fixes.

Target: 80/148 (54%). This means annotating ~58 additional papers.

Deliverables:
- modality, task, and evidence_level annotations for ~58 additional long-tail papers
- updated `metadata/paper_catalog.csv` rebuilt from source
- regenerated `metadata_completeness_report.json` confirming the new coverage

Annotation approach (manual, title+abstract based):
- For each unannotated paper, read the title from `papers_index.csv` and infer modality and task from title keywords
- Evidence level defaults to `primary` for papers that present a new method/architecture, `analysis` for ablation/survey/analysis papers, `review` for surveys
- Mark uncertain annotations explicitly in the notes field

Priority order for annotation:
1. Papers with "JEPA" in the title or paper_id (directly relevant to all modes)
2. Papers with arxiv IDs in the 2601-2605 range (most recent, highest chance of being relevant)
3. Remaining papers alphabetically

Why this is first:
- broader metadata coverage improves retrieval for ALL modes simultaneously
- it's the largest single quality lever remaining
- it unblocks richer citation candidates for reproduction and debug answers

### W2. Code Linkage Expansion

Objective: give reproduction_plan and debug_triage modes more concrete code evidence to reference.

Current state: 3/11 core papers have a code_id (I-JEPA → ijepa_main, MC-JEPA → eb_jepa_main, LeJEPA → lejepa_main). 6 core papers marked `no_known_local_code`, 2 marked `multiple_local_candidates` or `family_match_only`.

Target: 6/11 core papers with confirmed code_id.

Deliverables:
- investigate the 2 `multiple_local_candidates` / `family_match_only` core papers to see if a confident link can be established
- search for official or community repositories for the 6 `no_known_local_code` core papers
- update `metadata/code_index.csv` with any new archives
- update `metadata/paper_catalog.csv` with resolved links
- document findings in `docs/paper-code-linking-policy.md`

### W3. Reproduction & Debug Mode Quality

Objective: close the 10pp gap in executability and debug-hit.

Current state:
- reproduction_plan: 90% factual, 100% citation, 90% executable
- debug_triage: 100% factual, 100% citation, 90% debug hit

Likely failure modes (to diagnose first):
- reproduction plans may lack concrete environment/data/config details (vague steps)
- debug answers may rank hypotheses suboptimally or miss the primary failure mode

Deliverables:
- diagnosis of which specific prompts in the rp/dbg slices are failing and why
- targeted answer improvements for the failing prompts (similar to the RQ05/RQ07 workflow)
- judged rerun of the 10-prompt reproduction and debug slices

### W4. Writing Citation Mode Quality

Objective: close the 10pp factual correctness gap in writing_citation.

Current state: 90% factual, 100% citation, 0% hallucination.

The 10% factual error rate likely comes from over-generalization in drafted sections.

Deliverables:
- diagnosis of the failing writing_citation prompts
- improved claim scoping in drafted text (narrower claims, more conservative language)
- judged rerun of the 10-prompt writing_citation slice

### W5. Eval Regression Gate

Objective: ensure v1.3.0 changes don't regress research_qa.

Deliverables:
- rerun the full 40-prompt benchmark after all v1.3.0 changes
- produce v1.3.0 benchmark comparison against v1.2.1
- update CLAUDE.md eval reference table

## Suggested Execution Order

1. JEPA-301 + JEPA-302 + JEPA-303 (long-tail annotation, done together in one pass)
2. JEPA-304 (code links)
3. JEPA-305 + JEPA-307 + JEPA-308 (diagnose all three mode failures in parallel)
4. JEPA-306 (fix reproduction)
5. JEPA-307 fix phase (fix debug)
6. JEPA-308 fix phase (fix writing)
7. JEPA-309 (full40 regression + release)

## Release Risks

- The biggest delivery risk is that long-tail annotation is manual and tedious — ~58 papers takes focused effort. Mitigation: title-only annotation for obvious cases, deeper inspection only for ambiguous ones.
- The biggest quality risk is that metadata expansion alone doesn't improve reproduction/debug/writing scores. Mitigation: mode-specific diagnosis (JEPA-305/307/308) runs in parallel with metadata work.
- The biggest scope risk is trying to annotate all 148 papers instead of the 80-paper target. Mitigation: explicit 54% coverage target, stop when reached.

## Recommended Cut Line

If scope needs to shrink, keep JEPA-301 through JEPA-304 (metadata + code links) and JEPA-309 (release). This delivers a "v1.3.0 metadata release" with measurably better coverage even if mode-specific quality work is deferred to v1.3.1.
