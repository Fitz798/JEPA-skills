## RP09: "Select 5 JEPA papers suitable for reproduction"

### Prompt intent

Select 5 JEPA papers most suitable for reproduction from current metadata, explain the selection rationale, and provide evidence.

### What the baseline answer did

Selected I-JEPA, MC-JEPA, V-JEPA2, Audio-JEPA, and AMI as "beginner-friendly candidates." Mapped each to a paper evidence entry.

### Why it failed

1. **Factual correctness (0)**: The selection criterion is "suitable for reproduction," which implies code availability. But:
   - V-JEPA2 (`2025_v_jepa_2`) — `no_known_local_code` in catalog
   - Audio-JEPA (`2025_audio_jepa`) — `no_known_local_code` in catalog
   - AMI (`2022_a_path_towards`) — conceptual/philosophical paper, not reproducible
   - Only I-JEPA and MC-JEPA have confirmed local code archives
   - The answer didn't consult `metadata/code_index.csv` to verify code availability

2. **Executability (0)**: Since 3/5 selected papers lack local code, the reproduction plan is not executable. The action items (A1-A5) describe what each paper is about, not how to reproduce it.

### Fix strategy

- Query `metadata/code_index.csv` before selecting papers
- Replace V-JEPA2, Audio-JEPA, and AMI with papers that have confirmed local code (e.g., LeJEPA via `lejepa_main`, or use `ijepa_main` and `eb_jepa_main` as verified starting points)
- For papers without local code, explicitly note the gap and what the user would need to find externally
- Structure the answer as: paper → code availability → reproduction entrypoint → rationale

## DBG10: "Multi-GPU cross-machine training variance troubleshooting"

### Prompt intent

Systematic troubleshooting of environment factors when multi-GPU training shows large run-to-run variance across machines.

### What the baseline answer did

Four actions: align versions, verify dataset, compare launch flags, run smoke test. Evidence from protocol template and code index.

### Why it failed

**Debug hit (0)**: The fix order is directionally correct but too high-level. The notes say "lacks finer-grained experimental breakdown." Specifically:

- A1 ("Align versions") doesn't specify which versions matter most, or provide a comparison command
- A2 ("Verify dataset") doesn't specify checksum method or what to check
- A3 ("Compare launch flags") doesn't list the critical env vars (NCCL_*, CUDA_VISIBLE_DEVICES, etc.)
- No isolation protocol: how to narrow variance from "cross-machine" to "same-machine different GPU" to "same GPU different seed"
- Missing: the primary hypothesis should be ranked — NCCL/bus configuration is usually the top suspect for cross-machine variance, not driver versions

### Fix strategy

- Rank hypotheses with concrete probability assessment
- Add specific diagnostic commands for each hypothesis
- Include an isolation ladder: single-GPU baseline → multi-GPU single-node → multi-node
- List specific environment variables known to affect distributed training determinism
- Add verification checks after each fix step

## WRT04: "List 8 JEPA papers with contributions ordered by influence"

### Prompt intent

List 8 JEPA papers ordered by influence, explain each paper's specific contribution and usage/purpose.

### What the baseline answer did

Grouped 8 papers into 3 thematic claims (core lineage, stability branch, modality/planning branch). Each claim maps to 2-3 papers.

### Why it failed

**Factual correctness (0)**: The notes say "contribution descriptions exist but evidence is scattered per paper." The problem is structural:

- The prompt asks for individual paper descriptions ordered by influence, but the answer delivers grouped claims
- Papers within a group are not individually described — their specific contributions are collapsed into a group-level claim
- The ordering by "influence" is implicit in the grouping, not explicit
- Missing: why paper X is ranked #3 vs #4, what each paper's unique contribution is, and what specific use case each paper serves

### Fix strategy

- Restructure as an ordered list of 8 papers, each with: rank, paper_id, specific contribution, and use case
- Make the "ordered by influence" criterion explicit with brief rationale per paper
- Evidence mapping should be per-paper, not per-group
- Ensure coverage across the JEPA landscape: foundational (I-JEPA), video (V-JEPA), stability (Rectified LpJEPA), probabilistic (Var-JEPA), multimodal (VL-JEPA), analysis (feature norm/auxiliary tasks)
