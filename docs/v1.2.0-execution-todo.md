## Immediate Sequence

Execute these first without reordering:

1. `JEPA-202`
2. `JEPA-203`
3. `JEPA-204`
4. `JEPA-205`

Reason:

- `JEPA-202` creates the real data layer.
- `JEPA-203` prevents the new data layer from drifting.
- `JEPA-204` creates fields that retrieval and reproduction quality depend on.
- `JEPA-205` makes those fields useful.

## Active Milestone: M1 Data Backbone

Goal: move from template-only evidence handling to a real structured evidence asset.

Definition of done:

- `metadata/evidence_index.csv` exists
- evidence IDs are stable and unique
- locator quality is acceptable for core entries
- a validator fails on malformed rows
